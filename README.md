# Telegram Content Bot

Бот раз в минуту (или как настроено в `CHECK_INTERVAL`) проверяет Google-таблицу
и публикует посты по расписанию: сначала на русском, потом переводит на
украинский и публикует перевод во второй чат.

## Переменные окружения (задаются в Railway → Variables)

| Переменная              | Что туда вписать                                             |
|--------------------------|---------------------------------------------------------------|
| `TELEGRAM_TOKEN`         | токен бота от @BotFather                                     |
| `SPREADSHEET_ID`         | ID таблицы (из ссылки, без `https://` и без `/edit...`)      |
| `SHEET_NAME`             | название листа, например `Лист1`                             |
| `CHECK_INTERVAL`         | как часто проверять таблицу, в секундах (например `60`)      |
| `TIMEZONE`               | например `Europe/Kiev`                                       |
| `GOOGLE_CREDENTIALS_JSON`| весь текст JSON-файла сервисного аккаунта, одной строкой     |

## Структура таблицы

Первая строка — заголовки, дальше — данные:

`Дата | Время | Chat ID | Thread ID | Chat ID УКР | Thread ID УКР | Текст | Медиа | Медиа УКР | Статус`

- **Дата** — день недели текстом: `Понедельник`, `Вторник` и т.д.
- **Время** — например `9:00`.
- **Chat ID / Chat ID УКР** — ID групп (см. ниже, как узнать). Если пост нужно
  отправить в **несколько групп**, перечислите ID через запятую:
  `-1001111111111, -1002222222222`
- **Thread ID / Thread ID УКР** — ID темы (topic) внутри группы, если группа
  разделена на темы и нужно постить в конкретную (например "Важная информация").
  Если групп несколько (см. выше) — перечислите ID тем через запятую **в том же
  порядке**, что и группы в Chat ID: `10, 5`. Если у какой-то группы тем
  нет — просто оставьте пустое место между запятыми: `10, , 5`.
  Если тем нет вообще — оставьте всю ячейку пустой, бот пришлёт в общий чат.
- **Текст** — только на русском, бот сам переведёт.
- **Медиа / Медиа УКР** — ссылка на файл в Google Drive (не забудьте открыть
  доступ "Все, у кого есть ссылка", иначе Telegram не сможет скачать картинку).
- **Статус** — не трогайте, бот пишет сюда сам после публикации.
  Каждое воскресенье в 23:00 (по TIMEZONE) колонка очищается автоматически.

## Как узнать Chat ID и Thread ID

1. Добавьте бота в группу (сделайте администратором — надёжнее для тем).
2. Напишите тестовое сообщение внутри нужной темы (например "Важная информация").
3. Откройте в браузере: `https://api.telegram.org/botТОКЕН/getUpdates`
4. Найдите своё сообщение в JSON — там будет `"chat":{"id": -100...}` (это Chat ID)
   и `"message_thread_id": ...` (это Thread ID нужной темы).

## Как загрузить код на GitHub

1. Зайдите на https://github.com, войдите в свой аккаунт.
2. Нажмите **New repository**, дайте имя, например `telegram-content-bot`, нажмите **Create repository**.
3. На странице нового репозитория нажмите **uploading an existing file**.
4. Перетащите туда все файлы из этой папки (`bot.py`, `requirements.txt`, `Procfile`, `README.md`).
5. Нажмите **Commit changes**.

## Как запустить на Render.com (бесплатно)

Render держит бесплатным только "Web Service" (сервисы, отвечающие на HTTP) —
поэтому в код добавлен мини веб-сервер для health-check. Дальше нужен внешний
бесплатный "будильник", который будет стучаться в бота каждые ~10 минут,
чтобы Render не усыпил его из-за неактивности.

### Шаг 1. Создать Web Service

1. Зайдите на https://render.com, войдите (удобно через GitHub).
2. Нажмите **New +** → **Web Service**.
3. Подключите репозиторий `telegram-content-bot` (может потребоваться сначала
   разрешить Render доступ к вашим репозиториям на GitHub — так же, как
   делали для Railway).
4. Заполните поля:
   - **Name**: любое, например `telegram-content-bot`
   - **Region**: любой ближайший
   - **Branch**: `main`
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
   - **Instance Type**: **Free**
5. Нажмите **Create Web Service** — начнётся первая попытка деплоя (упадёт,
   пока не добавлены переменные — это нормально).

### Шаг 2. Добавить переменные окружения

1. В сервисе откройте вкладку **Environment**.
2. Добавьте те же 6 переменных, что описаны выше в разделе "Переменные окружения"
   (`TELEGRAM_TOKEN`, `SPREADSHEET_ID`, `SHEET_NAME`, `CHECK_INTERVAL`, `TIMEZONE`,
   `GOOGLE_CREDENTIALS_JSON`).
3. Сохраните — Render автоматически передеплоит сервис.

### Шаг 3. Не дать боту "заснуть"

1. Дождитесь, пока деплой станет **Live** — Render покажет публичный адрес вида
   `https://telegram-content-bot-xxxx.onrender.com`.
2. Зайдите на https://uptimerobot.com, зарегистрируйтесь бесплатно.
3. Создайте новый монитор: **+ New Monitor** → тип **HTTP(s)**.
4. В поле URL вставьте адрес вашего сервиса с Render (шаг выше).
5. Интервал проверки — **5 минут** (бесплатный тариф UptimeRobot это позволяет).
6. Сохраните.

Теперь UptimeRobot каждые 5 минут будет "стучаться" в бота, не давая ему
заснуть, а бот продолжит проверять таблицу и слушать сообщения в фоне.

### Шаг 4. Проверить логи

В Render откройте вкладку **Logs** — ищите:
- `Подключение к таблице успешно`
- `Слушатель входящих сообщений запущен`
- `Health-check сервер запущен на порту ...`

Если все три строки есть — бот полностью работает.

## Важно про доступ к таблице

Не забудьте открыть таблицу для email сервисного аккаунта (поле `client_email`
в JSON-файле) с правами **Читатель** — иначе бот не сможет её читать.
